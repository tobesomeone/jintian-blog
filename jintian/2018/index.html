webpack不仅仅是打包工具

2018-10-06
  1. 什么是webpack?
    官网的解释：webpack是现代javascript应用中的一种静态模块打包机。它对项目中涉及的各个模块进行映射，构建出一个依赖图表并打包成一个或多个文件。
    由此，我们可以获知：第一，webpack主要是javascript文件的打包机；第二：webpack适用于模块化的js文件，构建依赖图表；第三：webpack最后打包成一个或多个文件。
    关于第一点，webpack可以借助各种loader进行转化，比如css-loader,babel-loader可以将css文件，es6文件进行转化。

  1. webpack配置文件
    webpack配置有很多的选项，本博客主要关注常用的内容，比如：entry,output,loader,plugin,resolve,mode,devserver，devtool等，分别为入口，输出，加载器，插件，模块解析，开发模式，开发工具（development/production）。目前webpack4.0.0最新版本中，webpack可以省略entry,output配置文件，当然，手动配置webpack更好。由于这块内容非常简单，不做细数。

  2. 加载器loader
    首先，我们思考一下为什么需要loader? 我们回到webpack的最初定义，它是用来打包js文件，因此我们需要使用loader对项目中涉及的许多静态资源(非js资源)，比如图片，字体，css等文件格式进行预处理，然后使之打包成为依赖图的一部分。
    常用的loader：
    styling(样式文件预处理）：css-loader,style-loader,less-loader,sacc-loader,postcss-loader
    file（文本文件预处理）：file-loader,url-loader
    ts/js转换：ts-loader/awesome-typescript-loader,babel-loader
    html预处理：html-loader

  3. 插件plugins
    提供文件打包优化功能，资产管理，环境变量的注入，与loader的转换功能不同，plugins的重点在于优化。 如html-webpack-plugin，text-extract-webpack-plugin等。webpack提供了非常丰富的plugin接口，根据项目的需要进行相应的配置，使得webpack的功能非常的强大。
    常用的plugins：
    CommonsChunkPlugin 提取多个入口文件的公共部分（区别于DllPlugin的提取），这样一来，页面首次该文件只需加载编译一次，缓存放在客户端，优化了页面加载速度。在webpack4版本中，该插件被splitChunksPlugin替换，实现了更好的优化功能
    DllPlugin和DllReferencePlugin 将依赖的第三方库（比如angular,jquery等）独立打包成dll文件，因为这些库文件在开发中基本不会去改动，避免每次编译，将它独立出来，可以提高页面加载的速度
    ExtractTextWebpackPlugin 将css文件从打包文件中独立出来
    HotModuleReplacementPlugin 模块热替换，开发人员在web应用运行时，修改，增加或者删除部分源码模块内容之后，浏览器进行相应的内容更新但不需要重新加载页面。
    HtmlWebpackPlugin 将打包后的文件通过script标签添加到html文件中
    SourceMapDevToolPlugin 在控制台source栏源代码可视化，适用于开发环境
    CopyWebpackPlugin 拷贝文件或文件夹到最终的打包文件夹

  4. mode
    webpack分开发环境和生产环境，相应调用内置的优化工具。可通过两种方式进行配置

  5. resolve
    resolve可以设置webpack如何寻找模块文件，比如通过更改别名来加载绝对路径的模块；

  6. devServer
    通过使用 webpack-dev-server,提供一个简单的服务器功能，npm run start启动后就会自动打开浏览器，并且修改保存源码后，服务器会重新编译，重新加载，简化了在开发调试操作上的繁琐。支持HotModuleReplacementPlugin热加载，相互合作。

  7. devtool
    该选项主要是通过使用SourceMapDevTooPlugin插件，可以在调试阶段，通过浏览器的控制台具体定位代码编译错误的位置

  8. why webpack?
    说了这么多配置，那么，webpack到底好在哪儿呢？ 除了webpack，还有很多其他的打包工具，比如grunt，gulp等，为什么webpack更受欢迎呢？
    我们知道js的引用有两种：一种就是在html文档中以script标签一个个地引用js脚本；一种就是将所有js脚本合并为一个js文件，在html文档中以script标签引用。前者的问题是导致网络请求太多，后者问题是一个js文件太大，代码不好维护。
    前期，我们通过IIFE’s(立即执行函数表达式)这种模块化的场景来解决上述的问题，因此grunt, gulp等工具诞生了，但它们只是简单地将所有模块串起来。当你改动了一部分的js代码，或者应用只需要部分文件时，这些工具依然需要重新对所有文件进行打包，也就是说效率非常低下。
    而ECMAScrip6中倡导的模块化开发中，webpack非常好地避开了这些问题，提高了打包的效率。

docker相关知识

2018-08-09

#简介

  1. 什么是docker？
    NodeJs解释：将应用程序及其依赖以一种标准的单位进行打包。换言之，docker是一个快速部署的轻量级虚拟技术项目，开发人员可以一起打包程序和运行环境，成为一个镜像，只需要下载镜像文件就可以将程序跑起来，免去每次都安装各种依赖和环境的麻烦，还能够做到应用程序之间的隔离。粗糙地认为类似于虚拟机，但不同于虚拟机，

  2. 涉及的概念：镜像，容器，registry，

  3. 必备知识：linux命令行，bash

  4. 安装和配置： 去官网（http://www.docker.com/）下载对应的docker软件（本人使用windows版本）。 右键docker选择settings–》Docker Deamon–》 registry-mirrors 添加加速器连接 http://466ed505.m.daocloud.io/ （来自于阿里云或者网易云的docker加速器，提高docker下载速度）

  5. 本地运行: 打开命令行窗口，进行docker相关操作（最好跟着下面的指令在自己的电脑上敲一遍，看一下效果）
    5.1 基本操作：查看，启动和停止容器，下载镜像文件
    5.1.1 docker images //查看镜像列表
    5.1.2 docker ps //查看当前运行中的容器信息
    5.1.3 docker ps -a //查看所有容器信息，包括运行中和暂停的容器信息
    5.1.4 docker pull nginx //下载nginx镜像文件
    5.1.5 docker run -itd -p 10090:80 –name “my-nginx” nginx 

    //创建并启动nginx的新容器，并把虚拟机的80端口映射到本机的10090端口，在浏览器中可通过localhost:10090进行访问到nginx

    5.1.6 docker stop #containerId //停止指定的容器
    5.1.7 docker start #containerId //重启指定的容器，通过docker ps/docker ps -a 查看容器的

    5.2 在运行的容器内执行bash命令
    5.2.1 docker exec -it #containerId bash //在运行的容器中执行bash命令
    5.2.2 cat /etc/nginx/conf.d/default.conf //可以执行下述命令，如查看nginx镜像文件的配置文件
    5.2.3 apt-get update //
    5.2.4 apt-get install vim //安装vim
    5.2.5 vim /usr/share/nginx/html/index.html 打开步骤10看到的html文件，进行修改
    5.2.6 exit 退出

    5.3 提交修改后生成的新的镜像文件
    5.3.1 docker ps //查看
    5.3.2 docker diff
    5.3.3 docker commit -m “add: 修改了nginx首页的效果” containerId nginx2:1.0.1 //imageName自定义，tag也是 ,docker容器内的修改是暂存的，相当于 新的imgage,需要进行保存
    5.3.4 docker images //查看镜像列表，新添加了一个镜像nginx2

    5.4 删除容器操作
    5.4.1 dokcer stop containerId //停止指定容器
    5.4.2 docker ps -a //查看运行中的容器
    5.4.3 docker rm containerId //删除容器
    5.4.4 删除镜像：docker rmi imageId 前提必须先停止该镜像所在容器，然后删除容器，再删除镜像，若镜像被多个容器引用，需要强制删除,指令后面再 –force

    5.5 利用docker部署NodeJs应用，非常轻量和方便（还有问题，待补充）
    5.5.1 docker search node
    5.5.2 docker pull node //下载node镜像文件
    5.5.3 用nodejs编写一个简单的hello world后端文件app.js，docker images查看到node是否下载成功
    5.5.4 docker run -itd -p 10092:82 –name “my-node” node //为node镜像文件创建新的容器 //打开浏览器localhost:10092可以查看hello world效果
    5.5.5 docker cp app.js containerId:/home/hello //将该文件拷贝到制定的容器内，并指定路径

总结：使用docker的好处之一，就是方便学习新东西，用docker pull下载镜像文件，并在独立的容器内进行操作，用完后删掉容器和相应的文件，不影响本地
