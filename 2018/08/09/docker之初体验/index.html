docker相关知识

2018-08-09

  1. 什么是docker？
    NodeJs解释：将应用程序及其依赖以一种标准的单位进行打包。换言之，docker是一个快速部署的轻量级虚拟技术项目，开发人员可以一起打包程序和运行环境，成为一个镜像，只需要下载镜像文件就可以将程序跑起来，免去每次都安装各种依赖和环境的麻烦，还能够做到应用程序之间的隔离。粗糙地认为docker类似于虚拟机，但不同于虚拟机。两者的共同点：隔离和配置计算机资源；不同点，docker是在操作系统层面的虚拟化，而虚拟机是在硬件层面的虚拟化,下列图片很好地展示了两者区别

    docker特点：1. 安全 2. 轻量 3. 标准化

  2. docker涉及的主要概念：docker images，docker container，docker registry
    docker images是文件系统和参数的组合，docker里面所有文件都是基于镜像文件存在的，它是一种轻量，独立可执行的软件，内部包含了该应用程序运行所需的一切文件（如code, runtime, system tools, system libraries and settings），后续我们举例nginx镜像文件进行详细说明。
    docker container是将所有程序及其依赖打包后的一种标准的单元，应用程序通过此容器，可以在不同的环境之间保持快速可靠运行
    docker registry是一种无状态，高容量的服务器端应用，用来存储和分配docker images文件

  3. 安装和配置： 官网（http://www.docker.com/）下载对应的docker，并按照指示进行安装。为了加速连接，提高docker的下载速度，可以添加加速器（可采用阿里云或者网易云的docker加速器），具体操作如下图

  4. 安装好之后，就可以在命令行进行docker基本操作了，其指令和linux的操作类似，只是多了关键字docker,对于了解linux指令的开发人员来说，掌握基本指令其实是非常简单的事情。下列通过下载使用nginx进行详细说明

    1. 查看容器的基本情况
      1.1 docker images //查看镜像列表
      1.2 docker ps //查看当前运行中的容器信息
      1.3 docker ps -a //查看所有容器信息，包括运行中和暂停的容器信息

    2. 下载nginx镜像文件并创建容器
      2.1 docker pull nginx //下载nginx镜像文件
      2.2 docker run -itd -p 10090:80 –name “my-nginx” nginx
      //创建并启动nginx的新容器，并把虚拟机的80端口映射到本机的10090端口，在浏览器中可通过localhost:10090进行访问到nginx
      2.3 docker stop #containerId //停止指定的容器，通过docker ps -a 查看被停止的容器信息
      2.4 docker start #containerId //重启指定的容器，通过docker ps/docker ps -a 查看容器的

    3. 在运行的容器内执行命令 ，语法： docker exec [OPTIONS] CONTAINER COMMAND [ARG…]
      3.1 docker exec -it #containerId bash //在运行的容器中执行bash命令，成功进入后就可以使用bash命令进行相应操作
      3.2 cat /etc/nginx/conf.d/default.conf //如查看nginx镜像文件的配置文件
      3.3 vim /usr/share/nginx/html/index.html //比如编辑nginx下的html文件（apt-get update更新软件包，apt-get install vim安装vim）

    4. 由于docker容器内文件的修改是暂存的，需要进行提交保存。比如上述步骤3.3修改后的镜像文件进行提交保存
      4.1 docker ps //查看
      4.2 docker diff #containerId //文件的修改内容对比
      4.3 docker commit -m “comment” #containerId nginx2:1.0.1
      通过docker images查看镜像列表，新添加了一个镜像nginx2

    5. 操作完了，我们可以删除容器或删除镜像
      5.1 dokcer stop containerId //首先停止运行中的容器
      5.2 docker rm #containerId //删除容器
      5.3 docker rmi #imageId //删除容器内部的镜像文件
      注意：如果没有执行前两步操作，直接删除镜像文件会失败，提示先停止该镜像所在容器，再删除镜像，若镜像被多个容器引用，需要强制删除,指令后面再加 –force

  5. 总之，对于开发者来说，使用docker可以更方便地搭建和部署应用软件，在独立的容器内进行操作，用完后删掉容器和相应的镜像文件，不影响本地。后续文章将讨论如何利用dockerNodeJs应用,敬请期待。
